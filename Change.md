# Changes Made

This document summarizes the major changes made to Pactum during the current implementation pass.

## 1. Parallel Seller Negotiation

- Changed the negotiation flow from negotiating broadly or sequentially into negotiating with at most 3 of the most aligned seller agents.
- Seller selection is now based on matched suppliers and judged candidate quality.
- Added round-based parallel negotiation across selected sellers.
- Each selected seller keeps its own negotiation state, previous seller message, and active/rejected status.
- Negotiation now respects seller-specific negotiation style and the chosen buyer negotiation strategy.

## 2. Human Deal Controls

- Added a deal comparison pause after negotiation rounds.
- The user can now:
  - Accept one seller deal.
  - Reject all seller deals.
  - Send custom text as a counter-message.
- If the user sends a counter-message, negotiation continues with that message.
- If the user accepts one seller deal, all other seller deals are automatically rejected.
- The frontend deal modal now supports accept, reject all, and custom counter-message input.

## 3. Strict Product Matching

- Tightened product-family matching so requests only match exact product categories.
- GPU requests now reject accessories such as HDMI/VGA cables, adapters, risers, and cases.
- Laptop requests now reject sleeves, docks, chargers, cases, and laptop GPU components.
- Chair requests now reject mats, cushions, wheels, covers, and other accessories.
- Sensor requests now reject cables, brackets, covers, and similar non-sensor accessories.
- Sellers are excluded unless they have actual compatible products in the requested family.
- Partial sellers with only related accessories no longer enter negotiation.

## 4. Tavily External Supplier Fallback

- Added fallback handling when no internal inventory matches the requested product family.
- If the buyer asks for a product missing from local seller inventory, Pactum now uses Tavily enrichment instead of forcing an unrelated internal match.
- Tavily search results are converted into external supplier, product, and cluster rows.
- External candidates are clearly marked as estimated and should be verified.

## 5. Final fal Deal Card

- Reworked the fal integration so the final report can show a generated procurement deal thumbnail.
- The fal prompt is now generated from the chosen seller, product, price, delivery, warranty, technical status, and risk level.
- The prompt asks for a polished professional B2B deal-card thumbnail with:
  - Vendor name.
  - Fictional vendor logo mark.
  - Product name.
  - Offer details.
  - Pactum verified styling.
- Generated images are saved under `assets/deal_cards/`.
- FastAPI now serves static assets through `/assets/...`.
- The final report now displays the fal card image.
- If `FAL_KEY` or `FAL_API_KEY` is missing, Pactum falls back to the existing local `assets/fal_deal_card.png`.
- Added result metadata:
  - `deal_card_url`
  - `deal_card_prompt`
  - `deal_card_generated`

## 6. Backend API and Orchestration

## 11. Current Demo Behavior

The current app flow is:

1. Buyer enters a custom procurement prompt.
2. Backend extracts structured requirements.
3. Product matching keeps only exact product-family matches.
4. If no internal match exists, Tavily fallback creates external supplier candidates.
5. Candidates are judged and ranked.
6. At most 3 aligned sellers are selected.
7. The user chooses a negotiation strategy.
8. Selected sellers negotiate in parallel.
9. The user can accept, reject all, or send custom counter text.
10. Accepting one deal rejects the others automatically.
11. Offers are validated deterministically.
12. The final report displays the recommendation, audit summary, validation results, and fal deal card.
